FRAMEWORK TRADEOFF ANALYSIS
============================

DECISION 1: Blazor MAUI vs Swift/SwiftUI
-----------------------------------------
Initial decision: Swift/SwiftUI
Reversed: Yes — see Decision 2

Reason Swift won initially:
- WKWebView injects CSS/JS at document start (no flash)
- Full WKNavigationDelegate for hard URL blocking
- Smaller app size (~5MB vs ~50MB)
- Native performance

Why it was reversed:
- Team is on Windows — no Mac available for builds or device testing
- Swift requires Mac + Xcode for every build, sign, and deploy step
- No practical path to running the app without a Mac

DECISION 2: Swift vs Expo + React Native (final)
--------------------------------------------------
Decision: Expo + React Native (chosen)
Date: 2026-03-28

Why Expo wins for this team:
- Develop and test entirely on Windows
- Expo Go app on iPhone = instant testing, no Mac, no Apple account
- react-native-webview's injectedJavaScriptBeforeContentLoaded runs before DOM renders
  (equivalent to WKUserScript atDocumentStart — solves the flash problem)
- onShouldStartLoadWithRequest = hard navigation block (equivalent to WKNavigationDelegate)
- Same blocking capabilities as Swift, in TypeScript
- CI just needs a Node/TypeScript check — no macOS runner required for day-to-day

When a Mac is eventually needed:
- App Store submission only (EAS Build can handle this in the cloud)
- Not needed for development, testing, or Expo Go distribution

COMPARISON: Swift vs Expo/React Native for this app
-----------------------------------------------------

Factor                          | Swift/SwiftUI         | Expo + React Native
--------------------------------|-----------------------|---------------------------
Mac required to develop         | Yes                   | No
Mac required to test on device  | Yes                   | No (Expo Go)
Mac required for App Store      | Yes                   | Can use EAS Build (cloud)
Injection timing (no flash)     | atDocumentStart       | injectedJavaScriptBeforeContentLoaded
Hard navigation blocking        | WKNavigationDelegate  | onShouldStartLoadWithRequest
Language                        | Swift (new for team)  | TypeScript (more accessible)
Team's IDE (Rider)              | Poor Swift support    | Excellent JS/TS support
Cookie persistence              | WKHTTPCookieStore     | sharedCookiesEnabled prop
App size                        | ~5MB                  | ~25MB
Performance                     | Native                | Near-native for WebView use case
